### **‚öôÔ∏è Services Built (5 Major Services)**

#### 1. **TemplateService** (`template.service.ts`)
**Core Functionality:**
- ‚úÖ Create/update/delete templates
- ‚úÖ **`generateCaseFromTemplate()`** - Converts template ‚Üí case with tasks
- ‚úÖ Calculate task due dates from `durationDays` offset
- ‚úÖ Clone templates for versioning
- ‚úÖ Filter by country, tags, active status

**Business Logic:**
```typescript
// Automatically creates tasks based on template checklist items
// Each task due date = startDate + item.durationDays
// Maps ownerRole ‚Üí assigneeRole
// Handles dependencies between tasks
```

#### 2. **OnboardingCaseService** (`onboarding-case.service.ts`)
**Core Functionality:**
- ‚úÖ CRUD operations for cases
- ‚úÖ **`checkDay1Readiness()`** - Validates preboarding/provisioning complete
- ‚úÖ **Gate checking** - Blocks status transitions if requirements not met
- ‚úÖ Get overdue tasks, blocked tasks
- ‚úÖ Dashboard stats (active, at risk, completion rate)

**Business Logic:**
```typescript
// Gate Rule: Cannot move to IN_PROGRESS (Day 1) unless:
// - All PREBOARDING tasks are DONE
// - All PROVISIONING tasks are DONE
// Throws BadRequestException if gates fail
```

#### 3. **NotesService** (`notes.service.ts`)
**Core Functionality:**
- ‚úÖ Add/update/delete notes
- ‚úÖ **@mentions** support with notifications
- ‚úÖ Threading (replies to notes)
- ‚úÖ Pin/unpin notes
- ‚úÖ Visibility control (internal/shared for client portal)
- ‚úÖ **Auto-update parent object** (noteCount, lastActivityAt)
- ‚úÖ Search notes by content
- ‚úÖ Filter by author, object type, visibility

**Business Logic:**
```typescript
// When note is added:
// 1. Update parent task/case with noteCount++
// 2. Set lastActivityAt = now
// 3. Send notifications for @mentions (integrates with notification service)
```

#### 4. **CXOService** (`cxo.service.ts`)
**Core Functionality:**
- ‚úÖ Create customer cases from intake
- ‚úÖ **Auto-create 5 default workstreams** (Security, Legal, Technical, Billing, Training)
- ‚úÖ **`checkGoLiveGate()`** - Validates all 5 gates before enablement
- ‚úÖ Update gate checks (security, legal, billing, UAT, runbook)
- ‚úÖ Calculate completion percentage
- ‚úÖ Dashboard stats (TTL, SLA breaches, at-risk cases)

**Business Logic:**
```typescript
// Go-Live Gate Rule: Cannot move to ENABLEMENT unless:
// - securityApproved = true
// - legalApproved = true
// - billingApproved = true
// - uatApproved = true
// - runbookApproved = true
// Throws BadRequestException if any gate fails
```

#### 5. **RiskService** (`risk.service.ts`)
**Core Functionality:**
- ‚úÖ Add/update/delete risks
- ‚úÖ Get critical risks (CRITICAL/HIGH severity)
- ‚úÖ **Risk burndown** calculation with trend analysis
- ‚úÖ Get overdue risks (past target resolution date)
- ‚úÖ Calculate risk score (severity √ó probability)
- ‚úÖ Risk matrix for visualization

**Business Logic:**
```typescript
// Risk Score = severityWeight √ó probability
// Severity weights: LOW=1, MEDIUM=2, HIGH=3, CRITICAL=4
// Trend: Compare last 7 days vs previous 7 days
// Trend: improving | stable | worsening
```

```bash
  "jobTitle": "Software Engineer",
  "hiringManagerId": "MGR001"
}
```

- Task 1: Due Oct 18 (14 days before start)
- Task 2: Due Oct 27 (5 days before start, depends on Task 1)
